◆ spdk_nvmf_ctrlr_restore_migr_data()

int spdk_nvmf_ctrlr_restore_migr_data ( struct spdk_nvmf_ctrlr *  ctrlr,
const struct spdk_nvmf_ctrlr_migr_data *  data 
)

Restore the NVMe-oF controller state and configuration.

The API is experimental.

It is allowed to restore the data only when the nvmf subystem is in paused state.

To preserve thread safety this function must be executed on the same thread the NVMe-OF controller was created.

AERs shall be restored using spdk_nvmf_request_exec after this function is executed.

Parameters
ctrlrThe NVMe-oF controller
dataThe NVMe-oF controller state and configuration to be restored
Returns
0 on success or a negated errno on failure.

Save the NVMe-oF controller state and configuration.

The API is experimental.

It is allowed to save the data only when the nvmf subystem is in paused state i.e. there are no outstanding cmds in nvmf layer (other than aer), pending async event completions are getting blocked.

To preserve thread safety this function must be executed on the same thread the NVMe-OF controller was created.

Parameters
ctrlrThe NVMe-oF controller
dataThe NVMe-oF controller state and configuration to be saved
Returns
0 on success or a negated errno on failure.
